# Macrium Software Coding Challenge (MP3 variant)

## Task Description

The challenge requires reading an MP3 file from the command line and extracting key audio data, including frame count, sample rate, and average bitrate.

### Requirements

- Handle both constant bitrate (CBR) and variable bitrate (VBR) MP3 files
- Avoid parsing ID3v1/ID3v2 structures to prevent misinterpretation
- Focus on MPEG 1 Layer III format only
- Keep the solution concise and straightforward

### Error Handling

Solutions must gracefully manage errors including missing files, format corruption, and other issues preventing successful completion.

## Starting Resources

A Visual Studio starter project is available via the provided link. However, participants may create solutions from scratch if preferred.

**Visual Studio Project:** [mp3tasks.zip](https://updates.macrium.com/jobs/coding/mp3task.zip)

## Frequently Asked Questions

- **Assistance:** Limited hints and tips are available upon request
- **Research:** Internet resources are permitted; direct copying of existing solutions is discouraged
- **AI Usage:** "Yes, as with using the internet to help, we have no problems with using AI to assist"
- **Language Choice:** C (ANSI or newer) or C++ (C++98 or newer) are acceptable
- **Libraries:** Any standard library approach is acceptable (C Standard Library, C++, Win32, etc.)

Participants should be prepared to discuss their implementation during interviews.
